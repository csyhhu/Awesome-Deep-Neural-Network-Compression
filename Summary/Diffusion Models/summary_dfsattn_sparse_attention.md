# DFSAttn: Dynamic Fine-grained Sparse Attention for Efficient Video Generation

## 基本信息

- **论文标题**: DFSAttn: Dynamic Fine-grained Sparse Attention for Efficient Video Generation
- **作者**: Jie Hu, Zixiang Gao, Yutong He, Kun Yuan (北京大学)
- **会议**: ICML 2026
- **链接**: https://arxiv.org/abs/2605.23445
- **代码**: https://github.com/jessica-hujie/DFSAttn

## 核心问题

视频 Diffusion Transformer (DiT) 使用时空 3D Full Attention 实现高质量视频生成，但注意力计算的二次复杂度带来巨大开销。例如，HunyuanVideo 生成 129 帧 720p 视频在 H100 GPU 上需要约 30 分钟。

Block Sparse Attention 是一种常见的加速方法，通过块级稀疏化减少计算量。然而，DiT 中的注意力图具有**动态且细粒度**的稀疏模式，现有块稀疏方法在高稀疏度下质量严重下降。

## 核心观察与理论分析

### 观察 1: 注意力图的动态细粒度稀疏性
- 注意力模式在不同层和不同注意力头之间变化显著
- 重要的 token 交互分散在注意力图中
- 静态或粗粒度的稀疏掩码会丢失关键依赖

### 观察 2: 块稀疏注意力随去噪过程逐渐有效
- 在固定稀疏率下，注意力 Recall（保留的注意力质量比例）随去噪步数单调递增
- 早期去噪步骤中表征受噪声主导，注意力分布分散；后期逐渐集中

### 理论下界
作者提出了 Token 表征模型（假设每个块内的 token 可分解为块中心 + 块级语义漂移 + token 级扰动），推导出块稀疏注意力的 **Recall 下界**：

- 正确块选择的概率主要由**最小块间相似度差距**决定
- token 级噪声 σ² 和块级语义漂移 τ² 会降低选择可靠性
- **三个关键因素**：稀疏度预算 γ、块间相似度差距、块级语义多样性

## 方法: DFSAttn

DFSAttn 是一个 **免训练 (training-free)** 的动态细粒度稀疏注意力框架，包含三个核心设计：

### 1. 3D Hilbert 曲线 Token 重排序

**动机**: 扩大块间相似度差距。DiT 默认按帧的行主序展平 token，破坏了时空局部性。时空相邻的 token 在序列中可能相距很远。

**方法**: 
- 使用 3D Hilbert 空间填充曲线对视频 token 进行全局重排序
- Hilbert 曲线具有强局部保持特性，将 3D 视频空间投影到 1D 序列时保留时空邻域
- 重排序 → 块稀疏注意力 → 逆重排序恢复原始顺序

**效果**:
- 块内方差降低约 20%（queries: 1.22→0.98, keys: 1.25→1.02）
- 块级稀疏在重排序序列上等价于原始空间的**细粒度稀疏**
- 开销极小（12万 token 序列仅占 2% 运行时）

### 2. 层级块评分 (Hierarchical Block Scoring)

**动机**: 现有方法用粗粒度池化构建块级表示，当块内包含多个语义中心时，单一向量无法准确代表块的重要性。

**方法**:
- 将块进一步划分为子块（sub-block），子块内语义更一致
- 计算子块级别的注意力分数，聚合得到块级重要性分数
- 按聚合分数进行 Top-K 块选择（块大小 128，子块大小 16）

**效果**: 子块越小质量越好（16 > 32 > 64 > 无子块），且延迟几乎不变。

### 3. 自适应稀疏掩码缓存

**动机**: 
- 注意力 Recall 随去噪步数递增 → 后期可用更少预算达到同样精度
- 相邻去噪步的注意力掩码具有时间冗余性

**方法**:
- **自适应稀疏预算**: 稀疏预算随去噪步骤单调递减（初始 0.3，每 25% 步减 0.1，平均约 80% 稀疏度）
- **掩码缓存**: 每隔一定步数更新掩码（主实验每 12 步更新一次），其余步重用缓存掩码
- 注意：掩码可缓存复用，但**注意力输出每步重新计算**，确保表征动态演化不丢失

## 实例演示

下面用一个简化的具体例子，逐步展示 DFSAttn 如何在一个视频 DiT 的单个 Transformer 块中执行稀疏注意力。

### 场景设定

假设一个微型视频：**2 帧 × 2×2 空间**，共 `N = 2 × 2 × 2 = 8` 个 token，块大小 `B = 4`（共 `M = 2` 个块），子块大小 `B_s = 2`。

### Step 0: 原始序列的问题

DiT 默认按**帧优先 + 行主序**将 3D token 展平为 1D：

```
原始排列（Per-frame raster scan）:
┌──────────────────────────┬──────────────────────────┐
│ Frame 0                  │ Frame 1                  │
│ ┌────────┬────────┐      │ ┌────────┬────────┐      │
│ │ idx=0  │ idx=1  │      │ │ idx=4  │ idx=5  │      │
│ │ (h0,w0)│ (h0,w1)│      │ │ (h0,w0)│ (h0,w1)│      │
│ ├────────┼────────┤      │ ├────────┼────────┤      │
│ │ idx=2  │ idx=3  │      │ │ idx=6  │ idx=7  │      │
│ │ (h1,w0)│ (h1,w1)│      │ │ (h1,w0)│ (h1,w1)│      │
│ └────────┴────────┘      │ └────────┴────────┘      │
└──────────────────────────┴──────────────────────────┘
序列顺序: [0, 1, 2, 3, 4, 5, 6, 7]
```

**问题**: 同一空间位置跨帧的 token（如 `idx=0` 和 `idx=4`，都位于 `(h0,w0)`）在序列中间隔了 3 个位置。以 `B=4` 切分时，块 0 = `[0,1,2,3]`（纯 Frame 0），块 1 = `[4,5,6,7]`（纯 Frame 1）——**每个块内只有单帧信息，块间相似度差距很小**，导致块选择困难。

---

### Step 1: 3D Hilbert 曲线重排序

#### 什么是 Hilbert 曲线？

Hilbert 曲线是一种**空间填充曲线（Space-Filling Curve）**，由数学家 David Hilbert 于 1891 年提出。它的核心性质是：将高维空间中的点映射到一维序列时，**在曲线上相邻的点，在原高维空间中也相邻**。这一性质被称为**局部保持性（locality preservation）**。

```
2D Hilbert 曲线 (4×4):
┌───┬───┬───┬───┐
│ 0 │ 1 │14 │15 │    0 → 1 → 2 → 3 → ... → 15
├───┼───┼───┼───┤    每次移动只走相邻格子，
│ 3 │ 2 │13 │12 │    不跳远，连续遍历所有点
├───┼───┼───┼───┤
│ 4 │ 7 │ 8 │11 │
├───┼───┼───┼───┤
│ 5 │ 6 │ 9 │10 │
└───┴───┴───┴───┘
```

3D Hilbert 曲线是其在三维空间中的推广，在视频的 `(帧, 高, 宽)` 三个维度上同时保持局部性。

#### 3D Hilbert 曲线的构造（递归模式）

3D Hilbert 曲线按阶（order）递归构建。阶数 `n` 表示每维有 `2^n` 个离散点，共 `2^{3n}` 个点。

**1 阶 3D Hilbert 曲线（2×2×2 立方体，共 8 个点）**：

```
         w=1
        /|
       / |
      /  |
     ●───●  h=1
     |   |
  ●──┼───●
  |  ●───┼──●
  | /    | /
  ●──────●/  h=0
 /      /
●──────●     w=0
f=0  f=1

按曲线顺序展开:
  (f=0,h=0,w=0) → (f=0,h=0,w=1) → (f=0,h=1,w=1) → (f=0,h=1,w=0)
       ✻                                                    ⬇
  (f=1,h=1,w=0) ← (f=1,h=1,w=1) ← (f=1,h=0,w=1) ← (f=1,h=0,w=0)
```

**关键观察**：相邻点在三维空间中只差一个维度的一个单位，如 `(f=0,h=0,w=1)` → `(f=0,h=1,w=1)` 仅 `h` 轴移动了一步。

**高阶曲线的构造**：将 1 阶曲线作为"基元"，在 8 个八分象限中各放一个经过旋转/反射的基元，再用三段连接线串起来，反复迭代。论文图 3 展示了 2 阶曲线（4×4×4，共 64 个点）。

#### 在 DFSAttn 中如何使用

视频 VAE 编码后的 latent 张量形状为 `(f, h, w)`，如 HunyuanVideo 的 129 帧 × 40 × 64 ≈ 82,560 个 latent patch。DFSAttn 的处理流程：

```
Step 1: 计算 Hilbert 索引 (一次性预计算，之后复用)
  对每个坐标 (f_i, h_i, w_i) 计算其在 3D Hilbert 曲线上的位次
  → 得到排列 P: 原始 token 位置 → Hilbert 曲线上的新位置

Step 2: 重排序
  Q' = Q[P],  K' = K[P],  V' = V[P]
  （GPU 上这是一次高效的 gather 操作）

Step 3: 在重排序序列上执行块稀疏注意力
  按 B=128 切块 → 层级评分 → Top-K 选择 → SparseFlashAttn

Step 4: 逆重排序
  O = O'[P^{-1}]
  （GPU 上这是一次 scatter 操作）
```

#### 为什么比原始排列好？

以一个 129 帧的 720p 视频为例，假设一个 query token 位于视频中央（天空区域）：

```
原始排列 (Per-frame raster scan):
  同一帧的 token 按行排列 → 同一块内混入天空、建筑、地面
  → 块内语义混杂 → 均值池化后的向量模糊 → 无法准确判断该块的重要性

Hilbert 重排序:
  曲线优先遍历空间的邻近区域，再跨帧
  → 同一个块内全是同一片天空（跨帧但空间一致）
  → 块内方差降低 ~20% → 均值向量准确代表"天空"
  → 与"天空块"的相似度高，与"地面块"的相似度低
  → Top-K 选择更可靠
```

论文实验验证了不同重排序策略的效果：

| 策略 | PSNR | 说明 |
|---|---|---|
| Raster（默认行主序） | 27.794 | 基线 |
| Hilbert2D（逐帧 2D Hilbert） | 29.265 | 忽略了帧间连续性 |
| Block3D（分块后局部光栅） | 29.156 | 人为边界割裂 |
| **Hilbert3D** | **29.378** | 同时保持空间+时间局部性 |

#### 计算开销

Hilbert 遍历顺序只依赖于 latent 形状，**每轮推理只需计算一次**（甚至可以在模型加载时预计算）。对 12 万 token 的序列，重排序和逆重排序的 gather/scatter 操作仅占总运行时的约 **2%**，几乎可以忽略。

---

### Step 2: 层级块评分 (Hierarchical Block Scoring)

重排序后，序列 `[0, 1, 3, 2, 6, 7, 5, 4]` 被分为两个块（B=4）：

| 块索引 | 包含 token |
|--------|-----------|
| 块 0 | `[0, 1, 3, 2]` |
| 块 1 | `[6, 7, 5, 4]` |

**传统方法**（粗粒度池化）：对每个块的 Q、K 取均值 → 得到 2×2 的块分数矩阵，但一个均值向量难以代表块内所有 token 的语义。

**DFSAttn 的层级评分**（B_s=2）：

```
Step 2a: 将每个块划分为子块
  块 0: 子块 0a=[0,1], 子块 0b=[3,2]
  块 1: 子块 1a=[6,7], 子块 1b=[5,4]

Step 2b: 在子块级别计算 QK^T → 4×4 子块注意力矩阵 (更精细!)
          ┌──────┬──────┬──────┬──────┐
          │ 0a   │ 0b   │ 1a   │ 1b   │
  ┌───────┼──────┼──────┼──────┼──────┤
  │ 0a    │ 0.12 │ 0.08 │ 0.03 │ 0.01 │
  │ 0b    │ 0.07 │ 0.15 │ 0.02 │ 0.04 │
  │ 1a    │ 0.02 │ 0.01 │ 0.13 │ 0.07 │
  │ 1b    │ 0.01 │ 0.03 │ 0.09 │ 0.14 │
  └───────┴──────┴──────┴──────┴──────┘

Step 2c: 聚合到块级别
  S(块0→块0) = 0.12+0.08+0.07+0.15 = 0.42
  S(块0→块1) = 0.03+0.01+0.02+0.04 = 0.10
  S(块1→块0) = 0.02+0.01+0.01+0.03 = 0.07
  S(块1→块1) = 0.13+0.07+0.09+0.14 = 0.43

块分数矩阵:  ┌────────┬────────┐
              │ 0.42   │ 0.10   │
              ├────────┼────────┤
              │ 0.07   │ 0.43   │
              └────────┴────────┘

Step 2d: Top-K 选择 (γ=0.5, K=1)
  对查询块 0 → 选择键块 0 (分数 0.42)
  对查询块 1 → 选择键块 1 (分数 0.43)
  掩码 M = [[1,0], [0,1]]
```

层级评分的优势：子块级别的计算更接近真实 token 级交互，聚合后对块重要性的估计更准确。论文消融实验显示，子块越小效果越好（B_s=16 > B_s=32 > B_s=64 > 无子块）。

#### 为什么需要两层（子块 + 块），不能只用一层？

这里子块和块各司其职，并非冗余：

| | 用途 | 粒度 | 使用的数据 | 计算量 |
|---|---|---|---|---|
| **子块 (B_s=16)** | 估计/评分 | 细 | **池化后的均值向量** (压缩了 16 倍) | 便宜，`(N/16)²` 的 dense matmul |
| **块 (B=128)** | 实际注意力计算 | 粗 | **原始完整 token** (Q,K,V) | FlashAttention 高效执行 |

**为什么不能直接用子块做稀疏注意力？**

1. **内核不兼容**：FlashAttention 的块大小通常设计为 128。用 B_s=16 会导致大量无效的块边界处理，GPU 硬件利用率大幅下降。

2. **掩码开销膨胀**：子块数量是块的 8 倍（128/16），Top-K 选择、掩码索引和调度开销会显著增加，反而抵消稀疏带来的加速收益。

**设计本质**：

```
子块层 (仅用于估计):
  [池化 Q̂ (N/16 × d)] × [池化 K̂ᵀ] → 子块注意力矩阵 → 求和聚合 → 块分数

块层 (实际执行):
  根据块分数选出 Top-K 块 → FlashAttention 在选中块上计算真实的 QKᵀV
```

即**用廉价代理（池化子块）快速判断哪里重要，再用真实 token 在选中的块上精确计算**。如果子块再往下细分（如 B_s=4），收益递减且池化噪声增大。B_s=16 是精度和稳定性的平衡点。

---

### Step 3: 稀疏掩码缓存与自适应预算

假设去噪过程共 **50 步**，DFSAttn 的自适应策略：

```
去噪步数 t:  50 ──────────────────────────────→ 1
              ↑      ↑       ↑       ↑       ↑
稀疏预算 γ:  0.3    0.3     0.4     0.5     0.7
              │      │       │       │       │
掩码更新:    计算    复用    计算    复用    计算
              ↑──────↑───────↑───────↑───────↑
            t=50   t=38     t=25    t=13    t=1
           (更新间隔 Δ = 12 步，即每 ~25% 步更新一次)
```

- **早期 (t=50~38)**: 噪声大，注意力分散，用较高预算 γ=0.3 保证覆盖率
- **中期 (t=37~13)**: 逐渐收紧，γ=0.4→0.5
- **后期 (t=12~1)**: 注意力集中，γ=0.7 也能维持高 Recall

**整体平均稀疏度 ≈ 80%**，但质量明显优于固定 80% 稀疏策略。掩码每 12 步重新计算一次，中间复用缓存掩码；注意力输出始终每步重新计算，确保 token 表征的动态演化被完整保留。

---

### 端到端效果总结

以一个 129 帧 720p 视频为例（HunyuanVideo，约 12 万 token）：

| 阶段 | 操作 | 计算量 |
|------|------|--------|
| 重排序 | Hilbert 索引映射 (预计算) | ~2% 总时长 |
| 评分 | 子块注意力 (128/16=8 倍小于原生) | 轻量 |
| 稀疏注意力 | 仅计算 20% 的块对 | 大幅减少 |
| 逆重排序 | 逆向索引映射 | 可忽略 |

→ **端到端 2.1× 加速，PSNR 29.38，视觉质量接近 Full Attention**

## 关键亮点

1. **理论与实践的桥梁**: 从理论 Recall 下界出发，推导出三个可操作的设计方向
2. **巧妙的 Token 重排序**: 用 Hilbert 曲线实现"伪细粒度"稀疏，在不修改 GPU 内核的情况下获得细粒度效果
3. **免训练**: 无需微调或重新训练，即插即用
4. **正交性**: 可与步骤蒸馏、缓存等其它加速方法组合使用

## 与 SVOO 的对比分析

### SVOO 简介

SVOO（Sparse attention for Video generation via Offline sparsity profiling and Online co-clustering）核心发现是 **"注意力稀疏性是输入稳定的"**（Attention Sparsity is Input-Stable）——每个 Transformer 层的注意力稀疏模式是其固有属性,受不同输入的影响很小。

SVOO 采用两阶段范式：
1. **离线层稀疏度剖析**：用一个小校准集，逐层-逐头计算"注意力密度"（覆盖 95% 注意力质量所需的最小条目比例），拟合高斯分布后取上分位数作为保守估计，为每层分配不同的稀疏度预算
2. **在线双向共聚类**：交替执行"Query-aware 的 Key 聚类"和"Key-aware 的 Query 聚类"，显式建模 Q-K 耦合关系，使同一 query 块内的 token 具有相似的注意力偏好，同一 key 块内的 token 具有相似的被关注模式

### 核心差异对比

下表从八个维度系统对比两者如何产生 sparse attention：

| 维度 | SVOO | DFSAttn |
|:---|:---|:---|
| **核心洞察** | 注意力稀疏性是层的固有属性（输入稳定） | 注意力图具有动态细粒度稀疏模式，且块稀疏随去噪过程逐渐有效 |
| **是否需要校准数据** | **需要**离线校准集（少量 prompts） | **完全不需要**，全程在线 |
| **稀疏度分配策略** | **逐层差异化**固定预算（离线 profile 决定），不同层分配不同稀疏度 | **逐去噪步自适应**递减预算（初始 0.3，每 25% 步减 0.1），所有层共享同一调度 |
| **稀疏度决定依据** | 基于注意力密度的统计建模（高斯分布 + 上分位数） | 基于 Recall 下界的理论分析（三个因素） |
| **Token 分块方式** | **双向共聚类**：交替优化 Query 和 Key 的划分，显式建模 Q-K 耦合 | **3D Hilbert 曲线重排序**：基于时空局部性将 token 排列到 1D 序列，再等距分块 |
| **块重要性评估** | 质心内积粗估 + 阈值依赖选择策略（与 recall 和预算比较） | **层级子块评分**：子块级 QK 计算 → 聚合为块分数 → Top-K 选择 |
| **跨步复用策略** | 聚类结果跨去噪步**复用**（因输入稳定性保证分区不变） | 掩码跨步**缓存**（每 12 步更新一次），但注意力输出每步重算 |
| **GPU 适配** | Triton 自定义 kernel + FlashInfer 动态块大小 | 标准 FlashAttention kernel，固定块大小 B=128 |

### 方法差异的深层原因

两种方法虽然都是免训练的块稀疏注意力加速方案，但其设计哲学存在根本分歧：

#### 1. 对稀疏性来源的理解不同

- **SVOO** 认为稀疏性**来自模型参数**：层参数 $\mathbf{W}_Q\mathbf{W}_K^\top$ 决定了 logit 方差的分布，进而决定了注意力的集中程度。由于参数是固定的，所以稀疏性是"输入稳定"的。理论支撑是层稀疏度稳定性定理（Theorem 1），该定理证明了不同输入下的 logit 方差差异有界，且当 token 数 $n$ 很大时该界很小。
- **DFSAttn** 认为稀疏性**来自语义结构**：token 在时空上的语义邻近性决定了注意力模式。因此关键在于将语义上相似的 token 聚合到同一块中（扩大块间相似度差距），而非研究层的固有稀疏程度。理论支撑是 Recall 下界定理，揭示块选择正确性取决于块间相似度差距。

#### 2. 对分块方式的选择不同

- **SVOO 的共聚类** 是"注意力偏好驱动"的：谁对谁感兴趣，就把谁放在一起。这是一种**抽象语义**的视角，不显式利用时空位置信息。共聚类的好处是理论上能最优地匹配 Q-K 耦合关系，但代价是每步/每几轮的聚类计算开销较高。
- **DFSAttn 的 Hilbert 重排序** 是"时空邻近性驱动"的：位置相邻就把谁放在一起。这是一种**几何先验**的视角，利用了视频天然存在的时空局部性（邻近像素和邻近帧高度相关）。好处是计算极轻量（预计算索引映射，仅 2% 运行时），且天然诱导细粒度稀疏，但无法显式处理跨帧远距的语义关联。

#### 3. 对稀疏度分配的认识不同

- **SVOO** 关注的是**不同层的"容量"差异**：某些层注意力天生集中（如深层），可以高稀疏；某些层注意力天生分散（如浅层），需要低稀疏。这是一种**层的空间维度**上的差异化。
- **DFSAttn** 关注的是**不同去噪步的"需求"差异**：早期噪声大、注意力分散，需要高预算；后期表征清晰、注意力集中，可以低预算。这是一种**去噪时间维度**上的差异化。

这两种差异化本质上是**正交的**——理论上可以同时分配逐层和逐步的稀疏预算，这在 DFSAttn 的论文中被提及为可探索的未来方向，但当前两种方法各自侧重了一个维度。

#### 4. 校准需求的分歧

- SVOO 的离线校准**既有代价也有价值**：代价是需要在推理前跑一组 prompts 做 profiling（尽管论文声称只需少量 prompts）；价值是可以获得逐层-逐头的精细化稀疏度配置，将所有预算集中在"最需要的层"。
- DFSAttn 的零校准设计**牺牲了层差异化**（所有层同稀疏度），换来了**即插即用的便利性**。但通过自适应调度（逐步递减）和高效评分（层级子块），在不用校准的前提下达到了 SOTA。

| 指标 | SVOO | DFSAttn |
|:---|:---|:---|
| 模型 | Wan2.1-T2V-14B | Wan2.1-T2V-14B |
| 视频 | 81帧 720p | 81帧 720p |
| PSNR | ~29 dB | 22.37 |
| 稀疏度 | 未明确统一值 | 78.51% |
| 加速比 | 1.93× | 1.79× |
| 模型 | HunyuanVideo | HunyuanVideo |
| 视频 | 129帧 720p | 129帧 720p |
| PSNR | 未报告 | 29.38 |
| 稀疏度 | 未明确统一值 | 80.53% |
| 加速比 | — | 2.10× |

### 互为补充的视角

两种方法的核心贡献是**互补而非对立**的：

- **DFSAttn 的分块策略可以被 SVOO 采纳**：Hilbert 重排序是极低成本的"空间化"分块方式，可替代或增强共聚类的初始化
- **SVOO 的层差异化可以被 DFSAttn 采纳**：将 DFSAttn 的固定平均稀疏度替换为逐层不同的预算，进一步提升质量-效率平衡
- **两者可以组合**：校准出逐层稀疏度 + Hilbert 重排序 + 层级评分 + 自适应去噪步调度，有望达到更好的效果

## 局限与讨论

- 加速比受限于注意力模块占比，非注意力部分的计算仍是瓶颈
- 在极少数步数的蒸馏模型中，掩码复用的收益有限
- Hilbert 重排序的遍历顺序虽可预计算，但对非标准分辨率可能需要动态调整
