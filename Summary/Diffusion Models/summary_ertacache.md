# ERTACache: Error Rectification and Timesteps Adjustment for Efficient Diffusion

- **会议/期刊**: ICLR 2026
- **作者**: Xurui Peng\*, Chenqian Yan\*, Hong Liu\*, Rui Ma, Fangmin Chen, Xing Wang, Zhihua Wu, Songwei Liu†, Mingbao Lin† (* 共同一作, † 通讯作者)
- **机构**: ByteDance Inc., Rakuten Asia
- **Arxiv**: [2508.21091](https://arxiv.org/abs/2508.21091)

## 核心思想

提出 **ERTACache**，一个**基于误差校正和时间步调整的扩散模型缓存加速框架**。该工作首次系统性地将缓存引入的误差分解为两类——**特征偏移误差（Feature Shift Error）** 和 **步骤放大误差（Step Amplification Error）**——并分别设计对应的校正策略，在保持甚至提升生成质量的前提下实现 2× 以上的推理加速。

## 动机与背景

### 扩散模型推理瓶颈

扩散模型（尤其是视频扩散模型）推理极慢：
- **Wan2.1**：在 A800 上合成 81 帧 480p 视频需约 200 秒
- **CogVideoX**：50 步 480p 6 帧视频在 H800 上需 113 秒
- 根本原因：每步去噪都需前向传播大规模 UNet 或 Transformer，参数量随输入尺寸近乎平方增长

### 现有缓存方案局限

| 方法 | 缓存粒度 | 主要问题 |
|------|---------|---------|
| TeaCache | 层输出（残差） | 在线预测策略不稳定，预测值与实际误差不一致 |
| AdaCache / BAC / DiTFastAttn | Block/Attention Map | GPU 显存随帧数平方增长，长视频 OOM |
| FastCache / TokenCache / Duca | Token 级 | 与本文正交，可互补 |
| LazyDiT | 训练正则化 | 需要额外训练，成本高 |
| ICC | 低秩校准 | 无显式误差建模 |

### 核心发现

1. **跨 Prompt 一致性**：尽管扩散轨迹依赖于输入，但缓存复用模式和误差在不同 prompt 之间表现出强一致性——离线优化策略足以近似最优
2. **误差可分解**：缓存退化可分解为两种主导误差模式

## 方法

### 预备知识

**Flow Matching 框架**（SD3 / Flux 等主流方法）：
- 前向过程：\(x_t = (1-t)x_0 + t \cdot x_T, \quad t \in [0, 1]\)
- 真实速度场：\(u_t = x_T - x_0\)，模型 \(v_\theta(x_t, t)\) 学习预测该速度
- 推理（欧拉离散化）：\(x_{i-1} = x_i + \Delta t_i \cdot v_\theta(x_i, t)\)

**残差缓存机制**（沿用 TeaCache 做法）：
- 步 i+1 记录残差：\(\tilde{r} = v_\theta(x_{i+1}, t) - x_{i+1}\)
- 跳步时重建输出：\(\tilde{v}_i = x_i + \tilde{r}\)（避免完整前向传播）

### 误差分解理论

令 \(\tilde{v}_i = v_i + \varepsilon_i\)（缓存近似），连续 m 步缓存后的累积轨迹偏差为：

\[
\delta_{i-m} = \tilde{x}_{i-m} - x_{i-m} = \sum_{k=0}^{m-1} \underbrace{\Delta t_{i-k}}_{\text{Step Amplification Error}} \cdot \underbrace{\varepsilon_{i-k}}_{\text{Feature Shift Error}}
\]

- **特征偏移误差（Feature Shift Error）**：缓存残差与真实计算的直接偏差 \(\varepsilon_i\)
- **步骤放大误差（Step Amplification Error）**：时间积分区间 \(\Delta t_i\) 对误差的放大效应

ERTACache 框架通过三个组件分别针对这两种误差进行联合校正：

### 1. 离线策略标定（Offline Policy Calibration via Residual Error Profiling）

> 替代 TeaCache 的在线启发式预测，用离线搜索替代实时决策

**三阶段流程**：
1. **Ground-Truth 残差记录**：在小标定集上运行完整推理，记录所有时间步的真实残差 \(r^{gt}(x_i, t)\)
2. **阈值策略搜索**：遍历候选阈值 λ，基于相对 ℓ₁ 误差决定是否复用：
   \[
   {\ell_1}_{rel}(x_i, t) = \frac{\| \tilde{r}^{cali} - r^{cali}(x_i, t) \|_1}{\| r^{gt}(x_i, t) \|_1}
   \]
   若误差 < λ → 复用缓存；否则重新计算并刷新。最终得到缓存时间步集合 S
3. **推理时应用**：若 t ∈ S，复用缓存残差 \(\tilde{r}\)；否则计算 \(v_t\) 并刷新缓存

- λ 越小 → 缓存更新越频繁、质量越高；λ 越大 → 加速比越大、可能损失细节
- **关键洞察**：缓存模式和误差在不同 prompt 间高度一致 → 固定离线策略即可通用

### 2. 轨迹感知的时间步调整（Trajectory-Aware Timestep Adjustment）

> 纠正固定 Δt 下缓存重用导致的 ODE 轨迹漂移

- 引入校正系数 \(\phi_i \in [0, 1]\)，对缓存步动态缩小 Δt：
  \[
  \Delta t_i = \begin{cases} \Delta t_c, & i \notin S \\ \Delta t_c \cdot \phi_i, & i \in S \end{cases}
  \]
  其中 \(\phi_i = \text{clip}\left(1 - \frac{\| \tilde{v}_i - v_i \|_1}{\| v_i - v_{i+1} \|_1},\ 0,\ 1 \right)\)

- 剩余时间步预算动态重新分配：
  \[
  \Delta t_c = \frac{1 - \sum_{j=0}^i \Delta_{t_j}}{1 - i/T}
  \]

### 3. 显式误差修正（Explicit Error Rectification via Residual Linearization）

> 直接对缓存误差 ε_i 进行解析近似与补偿

- 假设误差可建模为：\(\varepsilon_i = \sigma(K_i \cdot \tilde{v}_i + B_i)\)（Sigmoid 单层卷积映射）
- 通过 Sigmoid 的一阶 Taylor 展开 \(\sigma(x) \approx \frac{1}{4}x + \frac{1}{2}\)，将非线性问题转化为线性回归
- 最小化 MSE 损失后推导出闭式解：
  \[
  K_i \approx 4 \cdot \frac{S_{v_i \varepsilon_i}}{S_{v_i v_i}}, \quad B_i \approx 4(\bar{\varepsilon}_i - \frac{1}{2}) - 4 K_i \cdot \bar{v}_i
  \]
  其中 \(S_{v_i v_i} = \sum(\tilde{v}_{ij} - \bar{v}_i)^2,\; S_{v_i \varepsilon_i} = \sum(\tilde{v}_{ij} - \bar{v}_i)(\varepsilon_{ij} - \bar{\varepsilon}_i)\)

- **推理时使用**：缓存步的输出替换为 \(v_i - \sigma(K_i v_i + B_i)\) 进行误差补偿
- K, B 参数可从小批量数据预计算，**额外开销近乎为零**：
  - 存储：+0%（OpenSora）/ +0.007%（Flux-dev）
  - 延迟：+0.14%（OpenSora）/ +0.45%（Flux-dev）

## 关键贡献

1. **形式化的误差分解理论**：首次将缓存引入的退化系统性地分解为特征偏移误差和步骤放大误差，为缓存加速提供了理论基础
2. **离线策略标定替代在线预测**：发现缓存模式在跨 prompt 间高度一致，用离线残差误差分析替代 TeaCache 的在线启发式预测，避免了预测不准和阈值敏感问题
3. **双维度联合校正**：时间步调整（纠正轨迹漂移）+ 显式误差修正（补偿缓存偏差），形成完整的误差感知缓存框架
4. **闭式解误差修正**：通过 Sigmoid 的 Taylor 线性化推导出 K、B 的闭合解析解，可从小批量数据预计算，推理开销可忽略（<0.5%）
5. **SOTA 性能**：在 3 个主流视频生成模型和 1 个图像生成模型上实现 1.55×−2.93× 加速，质量显著优于 TeaCache、PAB、Δ-DiT 等方法

## 讨论：残差是什么？

**问题**：论文中反复出现的"残差"（Residual）具体指什么？

**回答**：

1. **数学定义**：残差 = 模型在当前步的输出 − 当前步的输入 latent，即
   \[
   \tilde{r} = v_{\theta}(x_{i+1}, t) - x_{i+1}
   \]
   表示去噪网络对输入 \(x_{i+1}\) 所做的"增量变化"。

2. **为什么缓存残差而不是缓存原始输出？**
   - **自适应重建**：下一时刻的输入 \(x_i\) 可能不同于 \(x_{i+1}\)，但用 \(\tilde{v}_i = x_i + \tilde{r}\) 重建时，残差能自动适配新输入，保留结构信息
   - **节省存储**：残差量级通常小于原始输出，存储代价更低

3. **缓存流程**：正常计算步（i+1）→ 存残差 r̃；缓存复用步（i）→ 跳过模型前向，直接用 \(x_i + \tilde{r}\) 重建近似输出

4. **与 ResNet 残差的区别**：ResNet 的残差是网络**架构**层面的跳跃连接；本文的残差是**缓存策略**层面的概念——缓存的是模型输出相对于输入的变化量。两者不同层级的"残差"。

5. **残差与误差的关系**：缓存复用的本质是用上一步的残差来近似当前步的残差。这个近似引入的偏差 \(\varepsilon_i = \tilde{v}_i - v_i\) 就是 ERTACache 要修正的"缓存误差"。

## 讨论：是否只在 Flow Matching 上做实验？残差理解的常见误区

**问题1**：本文是否只在 Flow Matching 框架上做实验？

**回答**：是的，论文实验的 4 个模型（Open-Sora 1.2、CogVideoX、Wan2.1、Flux-dev 1.0）全部基于 Flow Matching 范式。但 ERTACache 的理论框架本身**不限于 Flow Matching**——它对 DDPM/DDIM 等经典扩散模型同样适用，底层原理（缓存引入的"特征偏移 + 步骤放大"两类误差）是通用的。只不过论文选择在当前主流的 Flow Matching 模型上验证。

**问题2**："残差是当前预估的速度减去前一个速度"，这个理解对吗？

**回答**：**不对，这是一个常见误解。** 正确的定义是：

\[
\tilde{r} = v_{\theta}(x_{i+1}, t) - x_{i+1}
\]

减的是 **\(x_{i+1}\)（当前步的 latent 输入）**，而非前一步的速度。三者的角色：

| 符号 | 含义 | 物理类比 |
|------|------|---------|
| \(x_{i+1}\) | 当前噪声 latent（模型**输入**） | 当前位置 |
| \(v_{\theta}(x_{i+1}, t)\) | 模型预测的速度场（模型**输出**） | 模型指示的移动方向 |
| \(\tilde{r} = v - x\) | 输出与输入的差值（**残差**） | 模型认为该"移动多少" |

结合 Flow Matching 的 ODE 更新式来理解：\(x_i = x_{i+1} + \Delta t \cdot v\)，残差 \(r = v - x\) 实质上是模型对当前噪声的"去噪增量"。缓存这个增量而非绝对输出，使得即使下一步 \(x_i \neq x_{i+1}\)，重建 \(\tilde{v}_i = x_i + r\) 仍能适配新输入。残差是 **model output minus current input**，**不是**两步速度之差。

## 讨论：残差缓存的归属与 ERTACache 三层创新的详细例子

**问题**：缓存残差这个做法是已有的还是作者提出的？

**回答**：残差缓存 **不是** ERTACache 原创。论文原文明确写道 *"Following prior work [Chen2024delta, Liu2025timestep]"*，即沿袭自 **Δ-DiT**（Chen et al., 2024，首次提出缓存残差加速 DiT）和 **TeaCache**（Liu et al., 2025，加入在线预测决定缓存时机）。ERTACache 的创新不在这套缓存机制本身，而在于**如何处理缓存引入的误差**。

| 层面 | TeaCache（已有） | ERTACache（本文新增） |
|------|:-:|:-:|
| 残差缓存机制 (\(r = v - x\)) | ✅ 提出 | 沿用 |
| 在线启发式预测 | ✅ | ❌ 替换为 ↓ |
| **① 离线策略标定**（决定哪些步缓存） | — | ✅ |
| **② 轨迹感知时间步调整**（纠正轨迹漂移） | — | ✅ |
| **③ 闭式解误差修正**（补偿缓存偏差） | — | ✅ |

### 用一个具体例子理解三层创新

假设在 **CogVideoX** 上以 T=50 步生成一段 48 帧 480p 视频。下面逐步跟踪 ERTACache 的完整工作流。

#### 第 0 层：残差缓存基础（已有方法）

正常推理时，每一步都要完整运行 Transformer 前向。TeaCache 的做法是：

```
步 49: 输入 x₄₉ → Transformer前向 → 输出 v₄₉ → 存残差 r̃ = v₄₉ - x₄₉
步 48: 输入 x₄₈ → [跳过前向！] → 重建 ṽ₄₈ = x₄₈ + r̃ → 更新 x₄₇ = x₄₈ + Δt·ṽ₄₈
步 47: 输入 x₄₇ → [跳过前向！] → 重建 ṽ₄₇ = x₄₇ + r̃ → 更新 x₄₆ = x₄₇ + Δt·ṽ₄₇  ← 误差累积！
步 46: 输入 x₄₆ → Transformer前向 → 输出 v₄₆ → 刷新缓存 r̃ = v₄₆ - x₄₆
...
```

**问题**：从步 49 到步 47 连续复用同一份残差 \(r̃\)，两步累积后偏差达到 \(\delta = \Delta t_{48}\varepsilon_{48} + \Delta t_{47}\varepsilon_{47}\)，视频可能出现闪烁或纹理模糊。

#### 第 1 层：ERTACache 的离线策略标定

ERTACache 不靠 TeaCache 那样的在线预测，而是在推理前做一次性的离线分析：

1. **准备标定集**：选取 20~30 个不同的 prompt，跑一次完整 T=50 步推理，记录每步的 ground-truth 残差 \(r^{gt}(x_i, t)\)。
2. **阈值搜索**：设定候选 λ = 0.08，逐步计算相对误差：
   \[
   \ell_{1,rel}(x_i, t) = \frac{\|\tilde{r}^{cali} - r^{cali}(x_i, t)\|_1}{\|r^{gt}(x_i, t)\|_1}
   \]
   假设结果是：
   ```
   步 49: ℓ1_rel=0.12 → >0.08 → 不缓存（重新计算）
   步 48: ℓ1_rel=0.03 → <0.08 → 加入缓存集合 S
   步 47: ℓ1_rel=0.05 → <0.08 → 加入缓存集合 S
   步 46: ℓ1_rel=0.15 → >0.08 → 不缓存
   ...
   ```
3. **得到缓存策略**：S = {48, 47, 44, 43, 41, 39, 38, 35, ...}，这是一个固定的时间步列表，对所有 prompt 通用。推理时直接查表决定是否缓存。

**核心优势**：全局搜索最优组合而非逐步独立决策，且完全消除在线预测的计算开销。

#### 第 2 层：轨迹感知的时间步调整

即使有了好的缓存策略 S，直接用固定 Δt_c = 1/50 = 0.02 仍然会导致 ODE 轨迹偏移。ERTACache 的做法是自适应缩小缓存步的步长：

```
正常步（i ∉ S）: Δt_i = Δt_c = 0.02          // 不做调整
缓存步 48:      已知 ṽ₄₈ 与真实 v₄₈ 偏差较大
                φ₄₈ = 1 - ||ṽ₄₈ - v₄₈||₁ / ||v₄₈ - v₄₇||₁ = 1 - 0.3 = 0.7
                Δt₄₈ = 0.02 × 0.7 = 0.014    // 步长从 0.02 缩小到 0.014
                
                剩余预算动态再分配:
                Δt_c = (1 - 0.014) / (1 - 1/50) ≈ 0.02012  // 微调后续步长
```

**直觉**：缓存步的输出不够准确 → 缩小这一步的"跨度"，走小步来减小误差放大。同时把缩掉的步长分摊到后续正常计算步上，保证总积分路径不变。

#### 第 3 层：显式误差修正（闭式解）

时间步调整只缓解了放大效应，但没有直接修正误差 ε_i 本身。ERTACache 进一步预先拟合出一个误差补偿量：

**离线阶段**：用标定数据计算每步的误差 ε_i = ṽ_i − v_i，然后通过闭式公式直接算出修正参数：
\[
K_i = 4 \cdot \frac{S_{v_i \varepsilon_i}}{S_{v_i v_i}}, \quad B_i = 4(\bar{\varepsilon}_i - 0.5) - 4 K_i \bar{v}_i
\]
（不需要训练，纯解析计算，20 条 prompt 数据即可稳定）

**推理阶段**：在缓存步 48 拿到重建输出 ṽ₄₈ 后：
```
未经修正:                 ṽ₄₈ = x₄₈ + r̃
ERTACache 修正后:          ṽ₄₈^{corr} = x₄₈ + r̃ + σ(K₄₈ · (x₄₈+r̃) + B₄₈)
                          = x₄₈ + r̃ + 补偿量
                          ≈ 真实 v₄₈
```

**效果**：以一个可忽略的加法运算（延迟 +0.14%~0.45%），直接补偿掉缓存带来的输出偏差。

## 讨论：用一句话理解三层创新

**问题**：本文的三个做法本质上是不是——用一些 val data，做三件事：确定哪些步可以复用、复用时的步长多大、复用时的补偿量多大？

**回答**：完全正确。三个组件都可以用"先拿 val data 离线算好，推理时直接查表/套公式"来理解：

| 组件 | val data 离线做什么 | 推理时输出什么 |
|------|---------------------|---------------|
| ① 离线策略标定 | 跑完整推理，算每步相对误差，按阈值 λ 筛选 | **缓存集合 S**：哪些步复用 |
| ② 轨迹感知时间步调整 | 算缓存步 ṽ_i 与真实 v_i 的偏差，预估 φ_i | **φ_i → Δt_i**：复用步走多"小" |
| ③ 闭式解误差修正 | 用 ε_i = ṽ_i − v_i 回归出 K_i, B_i | **补偿量 σ(K·ṽ+B)**：加回去多少 |

**② 和 ③ 的递进关系**：
- ② 是被动防御：输出不可靠 → 步子迈小一点，减小 Δt 对误差的"放大作用"
- ③ 是主动修正：直接预测偏差方向/幅度，把误差加回输出中
- 两者叠加：缩小 Δt 让 ③ 的修正有一个更小的误差底数，补偿起来更精准
