# Large Language Diffusion Models（LLaDA）阅读笔记

- **论文**：[arXiv:2502.09992](https://arxiv.org/abs/2502.09992)（v3，2025-10-18）  
- **题目**：Large Language Diffusion Models  
- **作者**：Shen Nie, Fengqi Zhu, Zebin You 等（人大高瓴、蚂蚁集团等）  
- **项目与代码**：[LLaDA Demo](https://ml-gsai.github.io/LLaDA-demo/)

---

## 一句话

**LLaDA（Large Language Diffusion with mAsking）** 是在「预训练 + 监督微调（SFT）」范式下**从零训练**的**掩码离散扩散语言模型**，用 Transformer 作为**掩码预测器**，在约 **8B** 规模上展示与强自回归基线（如 LLaMA3 8B）可比的**扩展性、上下文学习与指令遵循**，并在**逆向补全**等任务上体现扩散/双向建模的优势。

---

## 核心论点

业界常把大语言模型（LLM）能力与**自回归（AR）**划等号。本文认为更根本的是**生成建模原则**（最大化数据似然 / 最小化 KL），而非式子必须是从左到右的链式分解。扩散 Transformer 在视觉上的成功、以及「条件生成模型在结构一致的语言任务上」的共性，支持**非 AR** 路径也能获得可扩展能力与 ICL / 指令遵循类行为。与此同时，纯左到右生成与** reversal curse**（逆向推理弱）等现象，可归因于 AR 的归纳偏置。

---

## 方法概要

### 前向与反向

- **前向**：对干净序列 \(x_0\)，采样连续时间 \(t \sim U[0,1]\)，每个 token **独立**以概率 \(t\) 被替换为掩码 **M**，\(t=1\) 时全掩码。  
- **反向**：从全掩码响应出发，沿 \(t: 1 \to 0\) 多步迭代；每步用**掩码预测器** \(p_\theta(\cdot|x_t)\) **同时预测**当前所有 **M** 位置上的 token，再按扩散反向过程做 **remask**（使期望掩码比例与 \(s/t\) 一致）。

### 训练目标

在 **M** 位置上做交叉熵，并对 \(t\) 加权（文中式子含 **\(1/t\)** 因子）。该损失是模型分布负对数似然的**上界**，从而与（近似）**最大似然**、变分下界视角一致。作者强调与 BERT 固定掩码比、与 MaskGIT 等启发式目标（缺少 \(1/t\)、缺乏与 MLE 的严格联系）的差异。

### 架构与预训练

- **掩码预测器**：类 LLM 的 **Transformer，但不用因果 mask**（全序列可见；与 KV cache 不兼容，故采用**标准多头注意力**而非 GQA，并略调 FFN 以保持参数量级）。  
- **规模**：报告 **1B / 8B**；8B 在 **2.3T tokens** 上预训练，**4096** 上下文，约 **0.13M H800 GPU 小时**；**1%** 样本长度为 \(U[1,4096]\) 以增强变长能力。  
- **优化**：AdamW、Warmup-Stable-Decay 学习率调度等（正文给出具体数值）；作者称 8B 与 SFT **各跑一次、未做超参搜索**。

### SFT

- 对 prompt–response 对 \((p_0, r_0)\)，**只 mask response** \(r_0\) 得到 \(r_t\)，在 **M** 上对 \(r_0\) 算同样形式的加权交叉熵；与预训练目标形式兼容（拼接后等价于「干净 \(x_0\) + 只在回复段有 mask 的 \(x_t\)」）。  
- **4.5M** 对；batch 内对短样本在 response 末尾 pad **EOS**，训练时 EOS 当普通 token 参与 mask/损失，采样时去掉 EOS 之后内容以控制长度。

### 推理与评估

- **默认**：均匀离散时间步 + **低置信度 remask**（类似 MaskGIT；对比随机 remask 更好）。  
- **灵活性**：同一套权重上还可做 **AR 采样**、**block diffusion** 等而无需再训练；正文主表以**纯扩散采样**为准（整体更强）。  
- **条件 log 似然 / PPL 类评估**：使用与训练等价的另一形式（对固定数量 token 无放回 mask），方差更小（附录称 MC 次数可从 1000+ 降到约 128 量级更稳）。

## 主要实验结论（摘）

- **扩展性**：在与自建 **ARM 基线**相同数据与算力刻度（FLOPs，约 \(10^{20}\)–\(10^{23}\)）下，LLaDA 在 MMLU、GSM8K、HumanEval 等六项任务上的**整体趋势与 ARM 可比**；部分任务（如 MMLU、GSM8K）上扩散侧**斜率甚至更好**。  
- **Base 8B**：在统一复现协议下，**多数任务优于 LLaMA2 7B Base**，整体与 **LLaMA3 8B Base** 接近；数学、中文（CMMLU、C-Eval）等上有亮点，部分通用任务（如 BBH、HellaSwag）仍落后，作者归因于**数据分布与闭源数据差异**。  
- **Instruct 8B**：仅 **SFT**（无 RL 对齐）时，相对 LLaMA3 8B Instruct 等「SFT+RL」模型**略逊但整体差距不大**；案例展示**多轮对话、多语言翻译、约束生成**等指令遵循。附录指出 SFT 大量 EOS padding 会导致纯扩散采样过早结束，对 GSM8K/Math 等可对 **EOS 置信度置零** 等技巧；**Block Diffusion LLaDA** 在部分 instruct 指标上可高于表内纯扩散数（如 GSM8K 78.6 vs 69.4）。  
- **Reversal / 诗词补全**：自建 496 对名句任务上，LLaDA **前向**弱于 GPT-4o / Qwen2.5（数据与算力差距），但**逆向**补全显著更强，体现对 **reversal curse** 的缓解；作者直觉是**对 token 顺序无 AR 式偏置**。

---

## 局限与展望（文中自述）

- 生成长度仍是**超参**（虽对结果不极度敏感）。  
- 与同级 ARM 的「同架构同数据」对比在最大规模上受算力限制；未系统做 KV cache、专用位置编码等**系统级推理优化**。  
- **RL 对齐**、更强采样与可控性、多模态、Agent / O1 类后训练等留作未来工作。

---

## 个人小结

LLaDA 把**离散掩码扩散 + 似然上界训练**推到 **8B + 万亿 token** 量级，用**同一套数据与评估管线**与 AR 对照，论证「**核心 LLM 现象不必绑死 AR**」。工程上需接受**无 KV cache 的全注意力**与**多步解码**带来的成本，但换来**采样策略可调**、**双向结构**与在**逆向语言结构**上的实证优势。若后续在 RL、推理效率与数据规模上继续对齐顶级 AR 模型，结论会更有说服力。

## 问答与讨论记录（延伸阅读）

### Q1：remask 具体怎么做的？

**答：**

- **目的**：从噪声时间 \(t\) 走到更小的 \(s<t\) 时，模型会先把当前所有 **M** 位置并行预测一遍；但要让状态在统计上与「前向在 \(s\) 应有的 mask 比例」对齐，需要把一部分刚预测的 token **重新置为 M**，这一步即 **remask**。
- **比例**：正文写 **期望上** remask 掉 **\(\frac{s}{t}\)** 的「本步预测出的 token」，以匹配反向转移与 forward 过程（见 Sec. Inference）。
- **两种策略**：
  - **随机 remask**：原则上应对刚预测的 token 随机选取该比例重新 mask。
  - **低置信度 remask（默认）**：对刚预测的位置用 \(p_\theta\) 在 argmax token 上的概率作为置信度；**置信度最低的一批**被重新 mask（与 MaskGIT 同类；附录算法中有 \(n_{un}=\lfloor L(1-s)\rfloor\) 等与「保留多少已揭示 token」相关的实现细节，与时间表离散化一致）。

---

### Q2：反向采样就是生成（训练）过程吗？一般需要多少步？

**答：**

- **不是**。**训练**：对真实 \(x_0\) 抽一次 \(t\sim U[0,1]\)，独立 mask 得 \(x_t\)，一次前向在 M 位置上算加权交叉熵（含 \(1/t\)），用 SGD；**不在同一次反传里**从全 M 迭代到全文。
- **推理 / 生成**：从接近全 mask 的回复出发，沿离散时间多步「预测 → remask」直到接近无 mask，才是**反向采样**。
- **步数**：采样步数是**超参**，与质量–效率权衡相关；附录效率分析中在固定输出长度 256 下扫过 **32 / 64 / 128 / 256** 步（对应每步大致解出更多或更少 token）。评测设置示例：Base 上生成长度与步数常取 **1024**；Instruct 上步数可与「答案长度」任务相关地取 **3、64、256、512** 等（见附录评测说明）。**「一步训练 vs 多步推理」**：单次训练是「某个 \(t\) 上的一步去噪监督」；多步推理是把许多步去噪串成一条轨迹。（训练为何不多步展开见 **Q8**；「局部去噪任务」与整条轨迹是否算「同一件事」见 **Q9**。）

---

### Q3：训练过程是怎么进行的？

**答（流程摘要）：**

- **预训练**：\(t\sim U[0,1]\)；每个 token 以概率 \(t\) 独立变 M 得 \(x_t\)；Transformer **无因果 mask**，一次前向；只在 M 位置对 \(x_0^i\) 做 CE，并乘 **\(1/t\)**；目标为 NLL 上界。另有 1% 序列长度为 \(U[1,4096]\) 等细节。
- **SFT**：prompt 不动，**只 mask response** \(r_0\to r_t\)，条件为 \((p_0,r_t)\)，损失形式与预训练同构（mask 全落在回复段）。短样本 batch 内 **EOS pad**；训练时 EOS 参与损失，采样后截断。

---

### Q4：推理时 mask 变化方式有讲究吗？和 remask 的关系？

**答：**

- **有讲究**：mask 的演化必须与「离散化的 \(t\to s\) 转移」及前向独立 mask 的语义一致，否则采样链与训练假设不对齐。
- **remask 即实现手段**：**预测**解决「这一步填什么」；**remask** 解决「填完后仍应保留多少不确定性以对应下一步噪声水平」。低置信度 remask 是**启发式**，用「优先盖回不确定预测」近似随机 remask，实验中通常更好。

---

### Q5：直觉上是不是「每次把不确定的 token 又 mask 回去，直到能高置信生成」？

**答：**

- **大体对**，尤其适用于**低置信度 remask**：优先把当前最不靠谱的预测重新变 M，后续步在更好上下文中再填。
- **需两点限定**：（1）整体仍按**扩散时间表**走完整条链，不是「每个 token 独立循环到收敛」；（2）**理论上**更对齐的是**随机** remask，低置信度版是工程上更好用的变体。

---

### Q6：推理时生成长度怎么控制？

**答：**

- **硬上限**：**generation length** 为超参 = 采样开始时**全 mask 回复的槽位长度**（最多生这么长）。
- **软停止**：生成后丢弃 **首个 EOS 之后**的所有 token；SFT 里用 EOS pad 训练，使模型学会用 **EOS** 结束回复。
- **实务**：Instruct + 纯扩散 + 大量 EOS pad 时，可能出现 **EOS 过早过多**；论文在部分 benchmark 上对 **EOS 置信度置零** 等技巧缓解（见附录采样策略）。附录消融表明 Base 上对长度超参**不极度敏感**，但任务上仍需槽位足够大。

---

### Q7：生成速度是否主要取决于 remask？加速是否在于「减少 remask 数量」？

**答：**

- **主要瓶颈不是 remask 本身**：每步代价主要来自 **整段序列上的 Transformer 前向**（× 采样步数）；remask 多为排序与索引操作，相对可忽略。
- **加速更直接的方向**：**减少采样步数**、或 **每步提交更多 token**（少步 ↔ 每步多揭示，与附录效率实验一致）；块扩散 / semi-autoregressive、蒸馏、架构与系统优化等也是常见路线。
- **少 remask / 乱改 remask 比例**：更影响**转移是否正确与质量**，不能简单等同于「少算 FLOPs」；除非同时**减少前向次数**或改变并行提交策略，否则加速有限。

---

### Q8：为什么训练不采用多步（不在一次反传里模拟整条去噪链）？

**答：**

- **与目标一致即可**：损失是 NLL 上界 / 变分视角下的形式；对每个随机 \(t\) 在 **M** 上做加权 CE，已在期望意义上覆盖所有噪声水平。要学的是「给定 \(x_t\) 的一步去噪」；**推理时的多步**是把该能力按时间表与 remask **组合**成轨迹，训练不必在同一样本上展开整条链。
- **算力**：若每样本模拟 \(K\) 步「预测 → remask」再反传，前向与显存约按 \(K\) 放大，与 8B、万亿 token 级预训练不现实；现做法是每样本 **一次**随机 \(t\)、**一次**前向即贡献梯度。
- **与 Q2 对照**：训练不是「从全 M 迭代到全文」的同一条反传；推理才是多步链。

---

### Q9：能否说训练和推理「一样」——都是从某个被 mask 的状态恢复到干净序列？

**答（与讨论对齐）：**

- **在局部任务上可以这么说**：每一步都是「给定当前带 **M** 的序列，并行预测 **M** 位、推向干净 token」。训练和推理里**单步前向在干什么**是一致的，这也是**单 \(t\) 训练能支撑多步推理**的原因。多步 remask 等设计就是在尽量让这条链和训练假设一致。
- **在整条轨迹上仍要区分**：
  - **训练**：\(x_t\) 来自对**真实** \(x_0\) 按前向过程随机 mask；通常**没有**「模型预测 → remask → 再前向」的内层循环；监督始终对齐真 \(x_0\)。
  - **推理**：状态由上一步预测与 **remask** 演化，**多步**沿离散时间表走到接近无 **M**；中间分布会带上模型误差，与「从 \(x_0\) 一次 mask 得到的 \(x_t\)」可存在 **train–test gap**；论文中的时间表与 remask 设计意在让链与训练假设对齐。
- **一句话**：**语义上**都是去噪；**机制上**训练是「数据噪声、一步监督」，推理是「自举链、多步展开」。
