# RT-Lynx: Putting GEMM Sparsity in the Right Place for Diffusion Models

- **会议/年份**: arXiv 2026 (待定)
- **作者**: Xing Cong, Hanlin Tang, Kan Liu, Tao Lan, Lin Qu (Alibaba Group), Chenhao Xie (Independent Researcher)
- **链接**: https://arxiv.org/abs/2605.26632

---

## 1. 核心贡献

本文提出 **RT-Lynx**，首次在 DiT（Diffusion Transformer）上实现**无损 N:M 半结构化稀疏加速**（速度提升 1.55× 同时保持生成质量）。核心洞见是将稀疏化范式从**权重稀疏（Weight Sparsity）** 转向**激活稀疏（Activation Sparsity）**：

- DiT 的激活值天然稀疏（叠加机制导致每个 token 仅激活 5%~10% 的神经元），而权重接近高斯分布，缺乏 2:4 结构化稀疏的自然模式。
- 在激活上施加 2:4 稀疏化引入的误差远小于权重稀疏化，且对生成质量的影响更小。
- 设计了完整的端到端稀疏推理框架，包括：范数补偿稀疏化、LoRA 适应、选择性层跳过、以及高度优化的 CUDA 核。

---

## 2. 动机分析

### 2.1 权重天然不稀疏

- 权重值近似高斯分布，随机分布在整个值域范围内，没有明显的 2:4 结构化稀疏模式。
- 强制施加 2:4 稀疏化会移除大量关键参数，导致严重质量退化（FID 从 21.98 升至 51.63）。

### 2.2 激活天然稀疏（叠加效应）

- Transformer 存在 token 级别的叠加机制（Superposition），每个 token 仅激活少量 FFN 神经元。
- 激活值高度集中在零附近（如图 1(b)），2:4 约束主要消除接近零的值而非常用信息。
- 实验证明：激活稀疏化引入的相对误差显著小于权重稀疏化（图 1(c)）。

## 3. 方法

### 3.1 范数补偿稀疏化（Norm-Compensated Sparsification）

对激活 $\mathbf{X}$ 施加 2:4 Top-K 稀疏化后，对稀疏结果 $\tilde{\mathbf{X}}$ 进行 L2 范数重缩放：

$$S(\mathbf{X}) = s \cdot \tilde{\mathbf{X}}, \quad s = \sqrt{\frac{\|\mathbf{X}\|_2^2}{\|\tilde{\mathbf{X}}\|_2^2 + \epsilon}}$$

消除稀疏化引起的范数衰减，计算开销几乎为零。

### 3.2 LoRA 适应与微调

低幅值激活主要影响高频视觉细节（发丝、边缘、纹理），可通过低秩分支补偿：

$$\mathbf{Y} = S(\mathbf{X}) \cdot \mathbf{W}^\top + \mathbf{X} \cdot (\mathbf{L}_A\mathbf{L}_B)^\top$$

- $\mathbf{L}_A \in \mathbb{R}^{D_{\text{out}} \times R}$，$\mathbf{L}_B \in \mathbb{R}^{R \times D_{\text{in}}}$，Rank $R=64$
- 训练 2k 步即可收敛，参数增量仅 0.73%
- 损失函数：$\text{Loss} = \|\mathbf{X}\mathbf{W}^\top - (S(\mathbf{X})\mathbf{W}^\top + \mathbf{X}(\mathbf{L}_A\mathbf{L}_B)^\top)\|^2$

### 3.3 单流 DiT 的选择性层跳过

对于 FLUX 和 Z-Image 等单流（single-stream）架构中文本-图像混合层，稀疏化误差无法被 LoRA 完全补偿：
- **FLUX.1-dev**: 跳过单流路径中的 `attn.o_proj` 和 `mlp.down` 层
- **Z-Image**: 跳过单流路径中的 `attn.o_proj` 和 `mlp.up` 层

选择依据：对各层类型的相对 Frobenius 误差（RFE）进行全流程分析，跳过误差最大的层。

### 3.4 CUDA 核优化

解决了激活稀疏化面临的两个系统瓶颈：

| 问题 | 解决方案 |
|------|---------|
| 在线稀疏化开销高达 40%~59% 运行时间 | 将 Top-K 选择、压缩、格式转换融合为**单一 CUDA 执行路径**，直接在寄存器级别生成 SpTC 兼容的 2:4 激活布局 |
| LoRA 分支需要额外物料化和同步 | **交织执行**稀疏 GEMM 和密集 LoRA 计算，在芯片上完成 $\mathbf{Y}_s$ 到 $\mathbf{Y}_r$ 的累积，消除中间物料化 |

- 采用 streamK 风格的块并行流水线，最大化稀疏计算效率
- 在线稀疏化开销降至 <10%，Sparse GEMM 加速最高 1.88×

## 5. 关键洞见

1. **范式转移**：DiT 加速应聚焦于激活稀疏化而非权重稀疏化——激活天然具有叠加稀疏性，强结构化约束造成的伤害远小于权重。
2. **低秩补偿有效**：稀疏化主要影响高频细节，Rank=64 的 LoRA（参数仅增 0.73%）即可有效恢复。
3. **单流层敏感**：文本-图像共享的混合层对激活稀疏化特别敏感，需要选择性跳过。
4. **核融合是关键**：将在线稀疏化融合到 CUDA 执行路径中，将稀疏开销从 40%+ 降至 <10%，是实现实际加速的前提。

---

## 6. 局限性与讨论

- 端到端加速（1.2×）主要来自 Linear 层，Attention 等其他组件未加速，整体加速比受 Amdahl 定律限制。
- Z-Image 上权重稀疏化会导致灾难性失效（FID 360+），说明蒸馏模型的分布对齐对稀疏化极度敏感，RT-Lynx 在此场景下具有显著鲁棒性优势。
- 该方法与现有加速技术正交互补，可作为 DiT 推理栈中的基础组件。

## 7. 讨论与问答

### Q1: Token 级别的叠加机制是什么意思？

"叠加机制"（Superposition）来源于神经网络的**机械解释性（Mechanistic Interpretability）**领域，由 Anthropic 的 SAE（Sparse Autoencoder）系列工作率先系统性地提出，论文中也引用了该工作。

#### 核心问题

Transformer 的 FFN 层有 $D_{\text{out}}$ 个神经元维度（如 Qwen-Image 中 MLP 的 intermediate 维度为 12288），但模型在日常推理中需要表示和区分的概念远多于这个数量。那么，模型如何在有限维度中编码海量语义特征？

#### 叠加机制的答案

模型不是给每个概念分配一个独立神经元（那叫 one-to-one local representation，维度不够用），而是将**多个概念"挤压"到同一个高维空间中，每个概念用少数神经元的特定组合来表示**。这就是"叠加"——多个特征共享同一组神经元基底，通过激活模式的不同来区分。

#### 直观类比

想象一个房间有 100 盏灯（神经元），要表达 10000 个概念：

| 表示方式 | 策略 | 限制 |
|----------|------|------|
| **local**（一对多） | 每个概念点亮 1 盏特定的灯 | 最多只能表达 100 个概念 |
| **superposition**（叠加） | 每个概念点亮 5~10 盏灯的不同组合 | 理论上可表示 $C(100,5) \approx 7.5\times 10^7$ 个概念 |

Transformer 选择了叠加策略，结果是：**对任意一个 token（携带特定语义），仅有 5%~10% 的 FFN 神经元被显著激活，其余 90%+ 的激活值接近零**。

#### 在论文中的实证

论文的图 1(b) 清晰地展示了这一现象：
- 激活值的分布高度集中在零点附近，呈长尾分布
- 每一行（对应一个 token 的激活向量）仅有极少数通道有显著值（热力图上半部分中深色区域即为活跃神经元）
- 这与权重的准高斯分布（图 1(a)）形成鲜明对比

#### 对激活稀疏化的意义

叠加机制天然赋予激活**内在的结构化稀疏性**，当对激活施加 2:4 Top-K 稀疏化（每 4 个元素保留最大的 2 个）：

- **约 50% 的裁剪量实际上只作用于"本来就接近零"的 90% 低值元素**
- 真正活跃的 5%~10% 高值元素被保留
- 因此裁剪误差极小（图 1(c) 中激活稀疏的 RFE 远低于权重稀疏）

相比之下，权重值不具备这种天然结构——它们近似高斯分布，50% 的裁剪必然会移除许多量级中等、但信息不可替代的参数，导致严重质量退化。

**核心结论：叠加机制是激活稀疏化有效性背后的底层理论支撑——不是模型被"训练成稀疏的"，而是 Transformer 的表示策略天然导致稀疏激活模式。**

### Q2: 论文的核心方法流程可以如何概括理解？

论文的核心贡献可以归纳为一条清晰的逻辑链：

1. **发现（Why activation sparsity）**：DiT 的激活值天然稀疏（~5-10% 活跃），与 2:4 N:M structured sparsity 高度适配；相比之下权重呈高斯分布，稀疏化误差大得多。

2. **稀疏化（How to sparsify）**：在推理时对激活值在线进行 2:4 Top-K 裁剪——每连续 4 个元素保留量级最大的 2 个、其余置零。这是纯粹的推理时操作，不需要重新训练模型。

3. **补偿策略一：L2 范数补偿（Norm Echo）**：稀疏化后激活向量的 L2 范数会系统性衰减，因此用原始范数 / 稀疏化后范数作为缩放因子进行重缩放。这是一个**零成本、无参数**的操作，仅需计算逐行的 L2 范数比值。

4. **补偿策略二：LoRA 适应**：2:4 稀疏化虽然保留了量级最大的元素，但仍会丢失高频细节信息（低量级但仍有信息量的信号）。在 FFN 的线性层后插入 LoRA 分支（rank=64，仅 0.73% 额外参数），仅需 2k 步微调即可恢复这些信息。

5. **敏感层保护**：对 FLUX/Z-Image 等模型中的"单流层"（Single-Stream Blocks，text tokens 和 image tokens 在同一个 attention 中混合），稀疏化会破坏跨模态对齐，因此跳过这些层不做稀疏化，回退到 FP16 GEMM。

6. **工程优化**：将在线 Top-K 选择与 Sparse GEMM 融合为单个 CUDA kernel，将稀疏化开销从独立执行时的 40%+ 降至 10% 以内。

最终效果：
- **质量**：部分指标甚至超越原始 FP16 模型（稀疏化 + LoRA 相当于一种正则化 + 适应）
- **速度**：Linear 层加速 1.55×，端到端 1.2×
- **兼容性**：与蒸馏、量化、缓存、稀疏注意力等技术正交互补，叠加后最高可达 11.86× 端到端加速

### Q3: 本文需要重新训练吗？还是在原有模型上微调即可？

**不需要重新训练。** RT-Lynx 是一个 **post-training（后训练）方法**，整体流程分为两部分：

**无需训练的部分（推理时在线完成）：**
- 2:4 Top-K 裁剪：直接对激活值在线操作，不修改模型权重
- L2 范数补偿（Norm Echo）：纯数学重缩放，零参数、零训练

**仅需轻量微调的部分：**
- LoRA 适应：仅在 FFN 的 Linear 层旁挂 LoRA 分支，**不修改原始权重**
- 参数量仅 **0.73%**（rank=64），训练量极轻，仅 **2,000 步**即可收敛
- 本质是在冻结原始模型的前提下，用少量数据做针对性微调

这与 QLoRA / LoRA 的大模型微调范式类似——不是在重新训练模型，而是在已经训练好的模型上做轻量适应，让模型学会"容忍"激活稀疏化引入的微小信息损失。

**总结：核心稀疏化不需要任何训练；唯一的"训练"是 2k 步、0.73% 参数的 LoRA 微调，远非重新训练。**

### Q4: 如何理解 L2 范数补偿（Norm Echo）？

#### 问题的来源

2:4 Top-K 裁剪的操作是"每 4 个值中丢弃最小的 2 个"。无论被丢弃的值多接近零，**只要它不是完全为零，裁剪后激活向量的 L2 范数总会变小**：

$$||\tilde{x}||_2 \leq ||x||_2$$

这个系统性的"数值衰减"沿着网络层层传播，导致：
- 后续层的输入能量逐渐偏小
- 残差连接中的信号比例失衡
- 最终生成质量下降（细节模糊、对比度降低）

#### Norm Echo 的做法

极其简单——用**一个标量缩放因子**恢复范数：

$$\tilde{x}_{\text{echo}} = \tilde{x} \cdot \frac{||x||_2}{||\tilde{x}||_2}$$

其中 $x$ 是原始激活，$\tilde{x}$ 是 2:4 稀疏化后的激活。

**本质：既然裁剪丢掉了"能量"，就用总量不变的方式把能量"挪"到保留的元素上。**

#### 为什么叫 "Echo"（回声）？

原始激活的 L2 范数好比原始声音的音量。稀疏化后音量变小了（"声音被压缩"），Norm Echo 的作用是让稀疏激活"回声"出原始的音量——保持统计属性不变，信号不衰减。

#### 为什么有效？

| 特性 | 说明 |
|------|------|
| **零成本** | 只需计算两个 L2 范数再相除，纯数学运算，无参数 |
| **逐行独立** | 每个 token 的激活向量各自缩放，不影响 batch 内其他 token |
| **保留相对关系** | 是全局缩放（乘以标量），不改变向量内部的相对大小关系 |
| **弥补 90%+ 的低值** | 被丢弃的值虽小但数量多，累积起来范数损失不可忽略 |

#### 与 LayerNorm/RMSNorm/LoRA 的区别

| 方法 | 操作 | 粒度 |
|------|------|------|
| **Norm Echo** | 全向量 × 标量（恢复丢失的能量） | 每个 token |
| LayerNorm/RMSNorm | 均值归零 + 方差归一 | 每个 token |
| 逐元素补偿 | 每个元素单独调整 | 每个元素 |
| **LoRA** | 学习残差路径（恢复高频信息） | 整个层 |

Norm Echo 解决的是**宏观能量守恒**问题，LoRA 解决的是**微观信息补偿**问题，两者互补。
